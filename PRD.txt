1. 화면 상세 기획 (v3.2)
1.1. 스플래시 & 온보딩 화면
화면 구성:
스플래시 화면: 앱 실행 시 Ebeautything 로고와 브랜드 슬로건 "에뷰리띵 하나로 뷰티 비용 걱정 끝!"이 노출됩니다.
온보딩 화면: 서비스의 핵심 가치를 전달하는 2~3장의 슬라이드로 구성 (예: "내 주변 뷰티샵 찾기", "예약하고 포인트 받기", "포인트로 할인받기").
기본 기능: 앱 실행 시 스플래시 화면이 잠시 노출된 후 온보딩 화면(최초 실행 시 한 번만)으로 자동 전환되거나, 메인 화면(로그인 되어 있을 때)으로 바로 이동합니다. 온보딩 화면에서는 사용자가 "스킵" 또는 "다음" 버튼을 통해 자유롭게 이동할 수 있습니다.
연결 화면: 로그인/회원가입 화면
1.2. 로그인/회원가입 화면
화면 구성:
소셜 로그인: 간편한 로그인을 위해 카카오/애플/구글 로고 및 버튼을 통해 소셜 로그인 기능을 지원합니다.
회원가입: 이름, 성별, 생년월일, 전화번호 입력 필드, 전화번호 인증, 이메일, 추천인 입력 필드, 필수 약관 동의 체크박스, 가입 완료 버튼으로 구성됩니다.
회원 가입 완료: 회원 가입이 성공했음을 알리고 환영 메시지를 전달하는 화면입니다.
기본 기능: 사용자 전화번호 인증 및 계정 생성. 간편한 소셜 로그인을 우선으로 지원합니다.
연결 화면: 메인 화면 (로그인/가입 성공 시)
1.3. 메인 화면 (홈)
화면 구성:
상단: 현재 위치를 표시하며 (위치 선택 기능 제공), 뷰티샵 검색을 위한 검색창, 중요한 알림을 확인할 수 있는 알림 아이콘이 배치됩니다.
내 주변 샵 섹션: 사용자의 현재 위치를 기반으로 주변 뷰티샵 30개를 즉시 보여줍니다. 화면을 아래로 당겨 목록을 새로고침할 수 있습니다.
추천/인기 샵 섹션: 사용자의 관심사에 맞춰 추천되거나 인기 있는 뷰티샵을 캐러셀 또는 리스트 형태로 보여줍니다. (초기 노출 로직은 '2.1. 샵 노출 알고리즘' 참조)
카테고리별 샵 탐색: 네일, 속눈썹, 왁싱/속눈썹 문신, 헤어 (비활성화) 아이콘 및 텍스트로 쉽게 탐색할 수 있도록 제공합니다.
내가 찜한 샵: 사용자가 '찜하기' 해둔 샵들의 목록을 보여주는 섹션입니다.
이벤트/프로모션 배너: Ebeautything 자체 이벤트나 제휴 샵의 프로모션을 홍보하는 배너가 노출됩니다.
하단 내비게이션 바: 앱의 핵심 메뉴인 홈, 피드, 검색, 마이예약, MY 아이콘으로 구성되어 언제든지 쉽게 이동할 수 있습니다.
기본 기능: 사용자 위치를 기반으로 샵 정보를 제공합니다. 검색, 카테고리, 추천 등 다양한 경로를 통해 샵을 탐색할 수 있도록 돕습니다. 주요 서비스 메뉴에 접근할 수 있습니다.
연결 화면: 피드 화면, 검색 결과 화면, 샵 상세 정보 화면, 카테고리별 샵 목록 화면, 내가 찜한 샵 목록 화면, 알림 화면, 예약 내역 화면, 마이페이지
1.4. 샵 탐색 및 검색 결과 화면
화면 구성:
상단: 샵 검색을 위한 검색어 입력 필드와 함께, 결과를 효율적으로 찾을 수 있도록 필터/정렬 버튼이 제공됩니다.
검색 결과 목록: 각 샵은 이미지, 이름, 평점, , 거리, 간략한 서비스 정보와 함께 "공식 입점 샵" 또는 "미입점 샵"을 명확히 구분하는 아이콘을 표시합니다.
목록 보기 전환 버튼: 샵의 위치를 목록 형태로 자세히 볼 수 있도록 토글 버튼을 제공합니다.
기본 기능: 검색어(샵 이름, 서비스 종류 등)를 기반으로 샵을 검색하고, 가격대, 서비스 종류, 평점 등을 기준으로 필터링 및 거리순, 인기순 등으로 정렬할 수 있습니다. 목록 뷰 전환으로 편리한 샵 탐색을 제공합니다. 초기 기본 정렬 순서는 '2.1. 샵 노출 알고리즘'에 따릅니다.
연결 화면: 샵 상세 정보 화면
1.5. 샵 상세 정보 화면
화면 구성:
상단: 이전 화면으로 돌아가는 "뒤로가기" 버튼, 관심 있는 샵을 저장하는 "찜하기 아이콘", 샵 정보를 공유할 수 있는 "공유 아이콘"이 있습니다.
샵 정보: 샵의 특징을 보여주는 이미지 슬라이드, 샵 이름, 평점, , 주소, 편의정보, 영업시간, 전화번호 (바로 연결 가능), 샵 소개 글이 포함됩니다.
서비스 목록 섹션: 샵에서 제공하는 서비스 이름과 가격을 명확히 표시합니다.
예약/문의 버튼: 입점 샵은 "예약하기" 버튼이, 미입점 샵은 "문의하기"(전화연결) 버튼으로 변경됩니다.
가게 메시지 보내기: 샵이 카카오톡 채널 등 외부 연락처를 등록한 경우, 사용자가 해당 채널로 직접 메시지를 보낼 수 있는 "가게 메시지 보내기" 버튼을 노출합니다.
입점 샵 탭 메뉴: 입점 샵일 경우, [기본 정보], [시술 메뉴], [사진], [] 탭 메뉴가 생겨 각 섹션으로 편리하게 이동할 수 있습니다.
기본 기능: 샵에 대한 모든 상세 정보를 제공하여 사용자가 충분히 탐색하고 결정할 수 있도록 돕습니다. 간편한 전화 연결, 직접 메시지 문의, 예약 시스템으로의 진입점을 제공하며, 를 통해 다른 사용자들의 경험을 확인할 수 있습니다.
안내 문구: '예약하기' 버튼 주변에 "※ 타 플랫폼(전화, 네이버 예약 등)과의 중복을 피하기 위해, 사장님의 최종 확정이 필요합니다." 와 같은 안내 문구를 추가하여 사용자에게 사전 정보를 제공합니다.
연결 화면: 예약 요청 화면
1.6. 예약 요청 화면 (네이버 예약 UI/UX 참고)
화면 구성:
단계별 구성: 한 화면 내에서 스크롤하며 [서비스 선택] → [날짜/시간 선택] → [요청 사항] → [결제 정보] 순서로 정보를 입력합니다.
서비스 선택: 예약할 서비스를 체크박스로 선택합니다.
날짜/시간 선택: 캘린더와 시간 슬롯 UI를 통해 원하는 날짜와 시간을 선택합니다.
포인트 사용: 보유 포인트가 표시되고, 사용할 포인트 금액을 직접 입력할 수 있는 필드가 있습니다.
결제 정보: 예약금에 대한 총 예상 금액이 표시되며, 토스페이먼츠와 연동된 결제 수단 선택 옵션이 제공됩니다.
최종 확인 안내: 최종 예약 요청 버튼 상단에 "사장님 확인 후 예약이 확정돼요" 또는 "예약 요청 후, 샵의 확정 알림을 기다려주세요." 와 같은 문구를 명확하게 표시합니다.
기본 기능: 사용자가 원하는 서비스, 날짜, 시간을 편리하게 선택하고, 보유 포인트를 사용하여 할인을 받을 수 있습니다. 토스페이먼츠를 통해 간편한 예약금 결제를 지원합니다. 사용자가 제출하는 것은 '확정된 예약'이 아닌 '예약 요청'임을 명확히 인지시킵니다.
연결 화면: 결제 화면 (토스페이먼츠 연동), 예약 완료/대기 화면
1.7. 예약 내역 화면
화면 구성:
탭 메뉴: 사용자가 "예정된 예약"과 "지난 예약"을 쉽게 구분하여 볼 수 있도록 탭 메뉴가 제공됩니다.
예약 리스트: 각 예약 건은 샵 이미지, 샵 이름, 서비스명, 예약 날짜/시간, 예약 상태 등 핵심 정보를 포함합니다.
예약 상태 상세 설명:
예약 요청: '사장님 확인 중'이라는 부가 설명을 함께 표시하여, 사용자가 현재 예약이 확정을 기다리는 대기 상태임을 명확히 알 수 있게 합니다.
예약 확정: 샵에서 예약을 최종 승인한 상태입니다.
방문 완료: 서비스 이용이 완료된 상태입니다.
취소: 사용자 또는 샵에 의해 예약이 취소된 상태입니다.
예약 상세 보기 버튼: 각 예약 건을 클릭하면 상세 내용을 확인할 수 있도록 연결됩니다.
기본 기능: 사용자의 모든 예약 내역을 한눈에 확인하고, 예약 상태에 따라 직관적으로 분류하여 관리를 돕습니다.
연결 화면: 예약 상세 화면 (예약 확정 내역)
1.8. 마이페이지
화면 구성:
상단: 사용자의 프로필 이미지, 닉네임, 인플루언서 인증 마크 (해당 시), 보유 포인트가 표시됩니다.
메뉴 목록:
내 정보 관리 (프로필 수정)
포인트 관리 (포인트 내역 확인)
내가 추천한 친구들 (추천 현황 및 리워드 확인)
공지사항
1:1 문의 (에뷰리띵 카카오톡 채널로 이동)
자주 묻는 질문 (FAQ)
설정 (알림 설정, 앱 버전 정보)
로그아웃
기본 기능: 개인 정보 확인 및 수정, 포인트 이력 확인, 추천 현황 관리, 서비스 관련 정보 및 설정 변경 기능을 통합 제공합니다.
연결 화면: 내 정보 수정 화면, 포인트 내역 화면, 내가 추천한 친구들 화면, 공지사항 화면, 설정 화면
1.9. 포인트 관리 화면
화면 구성:
총 보유 포인트: 현재 사용 가능한 총 포인트 금액이 크게 표시됩니다.
포인트 적립/사용 내역 리스트: 날짜, 내용 (예: 예약 적립, 추천인 적립, 포인트 사용), 포인트 변동량 등 상세 내역을 리스트 형태로 제공합니다.
포인트 사용 가이드: 포인트 사용 기준 및 방법을 안내합니다.
기본 기능: 사용자의 포인트 현황을 투명하게 보여주고, 상세한 적립 및 사용 내역을 제공합니다.
연결 화면: 포인트 내역 화면 (상세 내역 확인)
1.10. 포인트 내역 화면
화면 구성:
총 보유 포인트: 현재 사용 가능한 총 포인트 금액을 다시 한번 표시합니다.
포인트 종류 탭/필터: "전체", "적립", "사용" 등 포인트 변동 유형별로 필터링할 수 있는 탭을 제공합니다.
상세 내역 리스트: 날짜, 내용 (예: XX샵 예약 적립, 김민지님 추천 리워드), 포인트 변동량 (+/- 표시), 변동 후 잔여 포인트를 상세하게 표시합니다.
페이징/더보기 기능: 내역이 많을 경우 페이지네이션 또는 "더보기" 버튼을 통해 추가 내역을 로드합니다.
기본 기능: 사용자의 모든 포인트 적립 및 사용 내역을 상세하고 투명하게 제공하며, 유형별 필터링 기능을 지원합니다.
연결 화면: 포인트 관리 화면 (뒤로가기)
1.11. 내가 추천한 친구들 화면
화면 구성(한달에 한번씩 업데이트되고 정산됨):
기간별 수익 조회: 은행 앱처럼 기간(시작일/종료일)을 설정할 수 있는 달력 UI를 제공합니다.
기간 설정 시, 해당 기간 내 추천으로 발생한 총 포인트 수익 (예: 총 +531,200 Point)과 수익에 기여한 친구들의 마스킹된 아이디 목록(예: dbgusg, gksthfal...)이 표시됩니다.
추천 통계: (조회 기간과 연동 또는 전체 기간) 총 추천 친구 수와 추천으로 받은 총 포인트를 요약하여 보여줍니다.
추천 코드 공유: 사용자의 고유 추천 코드를 표시하고, '복사하기' 또는 '공유하기' 버튼을 제공합니다.
추천 친구 목록:
추천한 친구의 아이디가 마스킹 처리되어 노출됩니다. (예: dbgusgh123 → dbgusg)
각 목록을 클릭(터치)하면 해당 친구를 통해 내가 받은 포인트 적립 내역이 상세히 펼쳐져 보입니다.
상세 내역 예시:
25/05/02 +150 point
25/06/12 +230 point
기본 기능: 추천인 시스템의 현황과 리워드를 직관적으로 관리합니다. 기간별/개인별 상세 내역 조회를 통해 추천 활동에 대한 명확한 피드백을 제공합니다.
연결 화면: 마이페이지 (뒤로가기)
1.12. 공지사항 화면
화면 구성:
공지사항 목록: 제목, 작성일이 표시되며, 중요 공지는 상단에 고정됩니다.
상세 내용: 공지사항 제목 클릭 시 아코디언 메뉴처럼 내용이 펼쳐지거나, 상세 페이지로 이동하여 전체 내용을 확인할 수 있습니다.
기본 기능: Ebeautything 서비스의 중요한 소식, 업데이트, 이벤트 등을 사용자에게 전달합니다.

2. 신규 정책 (v3.2)
2.1. 샵 노출 알고리즘
정의: 메인 화면, 검색 결과 등 샵 목록이 노출될 때 적용되는 기본 정렬 순서입니다.
초기 로직:
'입점 샵' 우선 노출: "공식 입점 샵" 아이콘이 표시된 제휴 샵을 최상단에 우선적으로 노출합니다.
최신 입점 순 정렬: '입점 샵' 그룹 내에서는 가장 최근에 입점 계약을 완료한 샵 순서대로 정렬하여 노출합니다.
목적: 초기 입점 샵에 대한 혜택을 강화하고, 사용자에게는 신규 샵을 적극적으로 알리기 위함입니다.
2.2. 인플루언서 기능
정의: 앱 내에서 높은 영향력을 발휘하는 사용자를 위한 리워드 강화 및 인증 제도입니다.
구현 방식:
추천인 시스템 연계: 회원가입 시 추천인 코드를 입력하는 기본 구조를 활용합니다.
자격 조건 및 지정 방식: Ebeautything 추천인 코드를 통해 50명 이상의 친구를 초대하고, 초대받은 친구 50명 모두가 1회 이상 서비스 결제를 완료했을 경우 인플루언서 자격이 자동으로 부여됩니다.
혜택:
리워드 강화: 인플루언서로 지정된 사용자는 일반 사용자에 비해 추천인 포인트가 2배로 적립됩니다.
인증 마크: 마이페이지 프로필 영역 및 향후 커뮤니티 활동 시 닉네임 옆에 공식 인플루언서임을 나타내는 인증 마크가 표시됩니다.
2.3. 서비스 지역
초기 범위: 서울시 전체
콘텐츠 구축: 신규 기능 개발 없이, 서비스 출시 전 미리 확보한 서울시 전체 샵 DB(데이터베이스)를 필터링하여 노출합니다.
향후 확장: 서비스 안정화 및 사용자 데이터 확보 후, 수도권 및 전국 주요 도시로 순차적 확장을 검토합니다.
2.4. 포인트 적립 정책 (수정)
결제 흐름:
1단계 (예약금 결제): 사용자는 앱을 통해 예약금을 우선 결제합니다.
2단계 (차액 결제): 시술이 완료된 후, 샵 점주가 앱을 통해 예약금을 제외한 차액을 청구합니다. 사용자는 이 청구에 따라 최종 결제를 진행합니다.
적립 시점: 사용자의 차액 결제가 최종 완료되는 시점에, 예약금을 포함한 '총 시술 금액'을 기준으로 포인트가 적립됩니다.
적립률: 총 시술 금액의 2.5%를 포인트로 적립합니다.
적립 한도: 포인트 적립은 총 시술 금액 기준 최대 30만원까지에 대해서만 적용됩니다.
예시 1: 총 시술 금액 20만원 결제 시 → 20만원의 2.5%인 5,000 포인트 적립
예시 2: 총 시술 금액 40만원 결제 시 → 30만원의 2.5%인 7,500 포인트만 적립
2.5. 포인트 사용 정책
사용 가능 시점: 포인트는 적립된 날로부터 7일(1주일)이 지난 후부터 사용 가능합니다.
정책 목적: 이 유예 기간은 토스페이먼츠-사용자-샵주 간의 정산 및 취소/환불 절차를 안정적으로 운영하고, 정산 타이밍의 정확성을 확보하기 위해 필요합니다.
실시간 포인트 시스템 구축에 대한 고려사항 실시간으로 포인트를 즉시 사용 가능하게 하는 시스템 구축도 기술적으로는 가능합니다. 하지만, 이러한 실시간 시스템은 사용자의 예약 취소, 결제 망 지연, 중복 사용 시도 등 예측 불가능한 변수들로 인해 복잡한 예외 처리가 필요하며, 이는 시스템 오류의 원인이 될 수 있습니다.
실시간 정산 및 포인트 사용 시스템을 도입하기 위해서는 아래와 같은 다양한 시나리오에 대한 정교한 정책 수립과 복잡한 로직 구현이 선행되어야 합니다:
예약 취소 시나리오: 예약 당일 취소, 노쇼(No-show) 발생 시 적립/사용된 포인트 처리 방안
포인트 회수 로직: 포인트 사용 후, 해당 포인트를 적립받았던 원 결제 건이 취소될 경우, 이미 사용된 포인트를 어떻게 회수 또는 처리할지에 대한 정책
결제 시스템 연동: 결제 시스템(토스페이먼츠)의 최종 정산 완료 시점과 앱 내 포인트 확정 시점을 동기화하는 구체적인 방안
오류 처리: 네트워크 오류 등으로 결제는 완료되었으나 포인트 지급에 실패했을 경우의 재처리 방안
이러한 요소들을 실시간으로 안정적으로 처리하기 위해서는 다수의 복잡한 예외 처리 및 정산 로직 개발이 필요하며, 이는 현재 MVP 개발 범위를 크게 초과하는 작업입니다.
또한, 7일 유예 기간 없이 즉시 정산이 진행된 후에는 오히려 예외 상황에 대한 사후 조치가 더욱 어려워질 수 있습니다. 이미 포인트가 사용되거나 정산이 완료된 이후에는 회수나 정정이 어렵기 때문에, 운영상의 리스크가 급격히 증가하게 됩니다.
따라서 현 단계에서는 7일 유예 기간을 유지하는 것이 서비스 안정성과 운영 효율성을 모두 고려한 최선의 선택이며, 추후 시스템이 고도화되면 실시간 정산 방안을 단계적으로 검토하는 것이 바람직합니다.
2.6. 예약 취소 및 환불 정책 (신규 추가)
정의: 사용자의 예약 취소 또는 노쇼(No-Show) 발생 시 예약금 환불에 대한 정책입니다.
환불 규정:
100% 환불: 고객이 예약 시간보다 충분히 이전에 취소하거나, 샵 점주가 예약을 취소하는 경우에는 예약금이 100% 환불됩니다.
환불 불가: 고객의 노쇼(No-Show) 또는 예약 시간에 임박하여 취소하는 경우에는 예약금이 환불되지 않습니다.
예외 처리: 정책에 따라 환불이 불가한 경우라도, 다양한 예외 상황이 발생할 수 있습니다. 이는 서비스 운영 과정에서 지속적으로 사례를 분석하고 정책을 보완해 나갈 예정입니다.
